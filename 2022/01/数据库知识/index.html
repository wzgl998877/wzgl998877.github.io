orcale中的decode() 函数

​ decode(条件,值1，返回值1，值2，返回值2,….值n，返回值n，缺省值)；

​ decode(字段或字段的运算，值1，值2，值3）

select  decode(COUPON_AMOUNT,'40.00','1','2'),COUPON_AMOUNT from T_CHN_WECHAT_LIST;--如果是40.00那么就是1否则就是2

​ 用来代替case when then else end

select 
   decode(name,'a',id,0) id_1,
   decode(name,'b',id,0) id_2,
   decode(name,'c',id,0) id_3 
from t_decode;

-- 相等于：case when then else end
select 
	case name when 'a' then id else 0 end as id_1,
    case name when 'b' then id else 0 end as id_2,
    case name when 'c' then id else 0 end as id_3 
from t_decode;

与order by连用

  1. order by 1,代表按第一个栏位排序，order by 加数字代表按第几个栏位排序。
  2. order by 中认为null是最大值，所以如果是ASC升序则排在最后，DESC降序则排在最前。
  3. order by 与 decode连用可以达到自定义排序的效果。
SELECT t.rowid rd, row_number() over (PARTITION BY t.chn_check_id ORDER BY decode(t.status, '2', '5', t.status), t.trans_time DESC) rn  FROM settle_user.t_check_trans_tmp_list t
WHERE t.chn_check_id IS NOT NULL -- 先根据chn_check_id 分区，然后根据状态和时间排序，其中如果status是2则按照5来排序，其他status就按照本来的值来排序，这里status的取值为0,1,2,3,4；所以5排在最后，所以当status为2时排在最后

case when

CASE WHEN用于SQL语句中的条件判断，Case具有两种格式，简单Case函数和Case搜索函数

--简单Case函数
CASE sex WHEN '1' THEN '男'
         WHEN '2' THEN '女'
         ELSE '其他' 
END
--Case搜索函数
CASE WHEN sex = '1' THEN '男'
     WHEN sex = '2' THEN '女'
     ELSE '其他' 
END

几个注意的点

  1. 在case when中若不加else，那么不满足条件就为null。
  2. 执行到end，如果end后还有语句会继续执行。
CASE WHEN sex = '1' THEN '男'
     WHEN sex = '2' THEN '女'
     ELSE '其他' 
END '太监'
-- 那么无论是什么都会变为太监 哈哈

orcale 中的序列

sequence是序列号生成器，可以为表中的行自动生成序列号，产生一组等间隔的数值(类型为数字)。其主要的用途是生成表的主键值，可以在插入语句中引用，在插入之前，获取序列号nextval值，然后进行插入。也可以通过查询检查当前值，或使序列增至下一个值。

创建序列

create sequence t_mng_user_user_id
increment by 1 -- 每次加几个
start with 1 -- 从几开始计数
nomaxvalue -- 不设置最大值
nocycle -- 一直累加，不循环
cache 10;

这里创建了一个序列，序列名称叫做t_mng_user_user_id 从1开始每次加一

使用序列

select t_mng_user_user_id.nextval from DUAL; -- 获得增加后的值
select t_mng_user_user_id.currval from DUAL; -- 获得当前值

使用场景

  • 不包含子查询、snapshot、VIEW的 SELECT 语句

  • INSERT语句的子查询中

  • INSERT语句的VALUES中

  • UPDATE 的 SET中

    例如：

update T_MNG_USER set USER_ID = t_mng_user_user_id.nextval where USER_ID = '2021030510101722';

集合函数

  1. union 并集运算
select USER_ID from T_MNG_USER union select USER_ID from T_MNG_USER_ROLE ;

注意：如果union前面是n列，那么后面也必须是n列，即union前后列数必须相同。而且查询结果的列名是按照union前面n列的名称命名

  1. INTERSECT 交集运算
select USER_ID  from T_MNG_USER intersect select USER_ID  from T_MNG_USER_ROLE where USER_ID = '2017112710000009';

注意：intersect前面列名和后面列名要相同

  1. MINUS 差集
select USER_ID  from T_MNG_USER where USER_ID in ('2017112710000009','2013061210007957') minus select USER_ID  from T_MNG_USER_ROLE where USER_ID = '2017112710000009';
